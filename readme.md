# ⚡SISTEMA DE FACTURACION ISI-INVOICE DOCUMENTO SECTOR ICE

Sistema desarrollado bajo la estructura de datos [vite](https://www.npmjs.com/package/vite)

Se debe tener instalado las extensiones `eslint` y `pretty` en su editor de codigo (vsCode, WebStorm, etc)

**Obligatoriedad el uso de YARN par la instalación de paquetes**

```bash
# Instalación global
npm i -g eslint
```

Comandos para formateo, dev. build

```bash
## Clonar el repositorio
git clone [repo]

## Instalar depedencias
yarn install

## Ejecuta los formateos y busca de errores en sitaxis
yarn lint

## Ejecución modo developer
yarn dev

## Compilacion
yarn build

## Vista previa del proyecto en produccion
yarn preview
```

#### FLUJO DE LOS SERVICIOS

Estamos usando la estructura de datos de [vite](https://www.npmjs.com/package/vite) para el desarrollo de este proyecto.

Ademas  de GraphQL, se esta usando el servicio de [Apollo](https://www.apollographql.com/docs/react/) para la gestion de las consultas y mutaciones.

Aquí se muestra un ejemplo de como se debe realizar una consulta a un servicio GraphQL de query y mutation.

> .[!NOTE]
> Usa una aplicacion como insomnia o postman para realizar las pruebas de los servicios

Example de una consulta a un servicio GraphQL

```graphql
query {
  getDocumentosSector {
    id
    nombre
    descripcion
  }
}
```

Example de una mutación a un servicio GraphQL

```graphql
mutation {
  createDocumentoSector(input: {
    nombre: "nombre"
    descripcion: "descripcion"
  }) {
    id
    nombre
    descripcion
  }
}
```
